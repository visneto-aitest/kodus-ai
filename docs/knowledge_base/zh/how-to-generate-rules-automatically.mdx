从头编写代码审查规则可能很繁琐。Kodus 可以分析团队的审查历史并根据发现的模式自动建议规则。

## 工作原理

1. 前往**代码审查设置** → **Kody 规则**
2. 点击**生成 Kody 规则**
3. Kodus 扫描代码库中 3 个月的代码审查历史
4. 识别反复出现的反馈模式和编码偏好
5. 生成适合您团队的规则建议

## 接下来

- 审查生成的建议 — 它们尚未激活
- 导入符合团队需求的规则
- 在激活之前根据需要自定义
- 只有导入的规则在代码审查期间执行

## 持续学习

初始设置后，Kodus **每周**生成新的规则建议，随着团队编码风格的演变。定期查看新建议。

## 要求

- 代码库中至少 **3 个月**的审查历史
- 定期的代码审查活动以获得更好的模式检测
- 具有适当权限的代码库访问

更多详情，请参阅[自动规则生成](/how_to_use/en/code_review/learning/kody_rules_generation)。
