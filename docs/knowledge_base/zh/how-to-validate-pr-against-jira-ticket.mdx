您写了代码，但它是否真的实现了 Jira 工单要求的所有内容？业务逻辑验证将您的 PR 差异与任务需求进行比较，标记任何缺失、不完整或不一致的内容。

## 工作原理

1. **连接 Jira 插件** — 在工作区的[插件](/how_to_use/en/code_review/plugins)页面中连接
2. 在 PR 中评论：

```
@kody -v business-logic https://kodustech.atlassian.net/browse/KC-1292
```

3. Kody 获取 Jira 工单（标题、描述、验收标准），读取 PR 差异并进行比较

## 您获得什么

Kody 生成结构化报告，包含：

- **状态** — 发现问题或合规
- **发现**及严重性级别：
  - **MUST_FIX** — 必需的业务规则未实现
  - **SUGGESTION** — 未涵盖的边缘情况或健壮性要点
  - **INFO** — 不阻碍合规性的有用观察
- **需求追溯** — 每个发现引用 Jira 工单中的确切需求
- **已验证需求** — 正确实现的内容，带有文件/行引用

## 自动验证

在审查选项中设置 `business_logic: true`（默认启用），Kodus 在每次 PR 审查期间自动验证关联任务 — 无需手动命令。

## 最佳效果提示

- 在 Jira 工单中添加**验收标准** — 具有明确标准的任务会获得逐条验证
- 如果 PR 范围与工单不匹配，Kody 会检测**范围不匹配**而不是产生误导性发现
- 您可以在解决发现后重新运行验证以确认合规

更多详情，请参阅[业务逻辑验证](/how_to_use/en/code_review/business_logic_validation)。
