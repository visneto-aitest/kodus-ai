---
title: "为什么我的代码审查被暂停了"
description: "了解 Kodus 自动暂停行为以及如何恢复自动代码审查。"
---

如果 Kodus 在之前正常工作后停止审查您的 PR，很可能是由于**自动暂停**功能。

## 什么是自动暂停

自动暂停防止活跃开发期间的审查垃圾信息。当 Kodus 检测到快速连续推送时（默认：15 分钟内 3 次推送），它会暂停审查直到活动稳定。

这是一个功能，不是 bug — 它避免了在您仍在积极推送更改时用多次审查淹没 PR。

## 如何恢复

- **等待**冷却期过后再推送
- **评论** `@kody start-review` 立即触发审查
- **更改节奏**为"自动"（如果您总是希望每次推送都审查）

## 更改行为

前往**代码审查设置** → **通用** → **审查节奏**：

| 模式 | 行为 |
|------|------|
| **自动** | 审查每次推送 — 不暂停 |
| **自动暂停** | 在快速推送时暂停（默认） |
| **手动** | 仅在您评论 `@kody start-review` 时审查 |

更多详情，请参阅[通用配置](/how_to_use/en/code_review/configs/general)。
