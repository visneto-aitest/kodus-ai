---
title: "Cómo verificar el código contra especificaciones de Google Docs"
description: "Valida la implementación de tu pull request contra especificaciones escritas en Google Docs."
---

Las especificaciones y requisitos de producto a menudo viven en Google Docs. Kodus puede obtener el contenido del documento y compararlo con el diff de tu PR para verificar que la implementación coincide con la especificación.

## Cómo funciona

1. **Conecta el plugin de Google Docs** en la página de [Plugins](/how_to_use/es/code_review/plugins) de tu espacio de trabajo
2. Asegúrate de que el documento sea accesible para la cuenta conectada
3. Comenta en tu PR:

```
@kody -v business-logic https://docs.google.com/document/d/1abc123/edit
```

4. Kody obtiene el documento, extrae los requisitos y los compara con el diff del PR

## Qué obtienes

Kody produce un informe de validación estructurado:

- **Hallazgos** con severidad: MUST_FIX, SUGGESTION o INFO
- **Trazabilidad de requisitos** — cada hallazgo cita texto del documento de Google
- **Requisitos Verificados** — lo que fue implementado correctamente

## Consejos

- Mantén las especificaciones estructuradas con secciones claras y requisitos numerados para mejores resultados
- También puedes pegar el contenido de la especificación en línea:

```
@kody -v business-logic
## Requisitos
1. Los usuarios con rol "admin" pueden eliminar cualquier comentario
2. Los usuarios solo pueden editar sus propios comentarios dentro de las 24 horas
3. Los comentarios eliminados muestran el marcador "[eliminado]"
```

- Divide los documentos grandes en secciones y valídalas individualmente para obtener feedback más enfocado

Para más detalles, consulta [Validación de Lógica de Negocio](/how_to_use/es/code_review/business_logic_validation).
