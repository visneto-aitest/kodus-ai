title: "Como configurar revisão de código com IA no Azure DevOps"
description: "Instale e configure o Kodus para repositórios Azure DevOps e receba revisão de código automatizada com IA em cada pull request."
---

O Kodus integra-se com o Azure DevOps via webhooks. Uma vez configurado, cada pull request é revisado automaticamente.

## Passos

1. **Crie uma conta** em [kodus.io](https://kodus.io) ou [hospede o Kodus por conta própria](/how_to_deploy/pt-br/deploy_kodus/generic_vm)
2. **Conecte o Azure DevOps** — autorize durante a configuração e configure seu Personal Access Token
3. **Configure os webhooks** — siga a configuração guiada para habilitar eventos de pull request
4. **Selecione os repositórios** — escolha quais repositórios Azure DevOps o Kodus deve revisar
5. **Abra um pull request** — o Kodus o revisa automaticamente

## O que você recebe

- Revisões automatizadas em cada PR
- Comentários postados diretamente no pull request
- Regras personalizáveis e controles de sugestões
- Validação de lógica de negócio contra itens de trabalho vinculados

## Configuração personalizada

Coloque um `kodus-config.yml` na raiz do seu repositório para personalizar o comportamento por repositório. O arquivo substitui as configurações da interface automaticamente.

## Usando um Personal Access Token

O Azure DevOps requer um PAT para que o Kodus acesse seus repositórios e poste comentários de revisão. Certifique-se de que o token tem os escopos adequados para leitura de código e acesso a comentários de PR.

Para o guia completo de configuração, consulte [Azure DevOps Webhooks](/how_to_deploy/pt-br/platforms/azure_devops/azdevops_webhook) e [Azure DevOps PAT](/how_to_use/pt-br/code_review/general_config/azure_devops_pat).
