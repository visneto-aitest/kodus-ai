---
title: "AIコードレビューがトリガーされない理由"
description: "KodusがプルリクエストをAutomaticにレビューしない場合の一般的な原因と修正方法。"
---

KodusがPRをレビューしていない場合は、以下の一般的な原因を確認してください：

## 1. 自動レビューが無効になっている

自動レビューが有効になっていることを確認します：
- **コードレビュー設定** → **一般** に移動します
- **自動レビュー**がオンになっていることを確認します
- または `kodus-config.yml` で確認します：

```yaml
automatedReviewActive: true
```

## 2. Webhookが設定されていない

KodusはPRが開かれたり更新されたりしたときを知るためにWebhookが必要です：
- Gitプロバイダーのwebhook設定に移動します
- KodusのWebhook URLが登録されてアクティブであることを確認します
- 配信失敗の試行を確認します

## 3. PRのタイトルが無視キーワードに一致している

PRのタイトルに無視キーワードが含まれている場合、Kodusはレビューをスキップします：

```yaml
ignoredTitleKeywords:
  - "release"
  - "WIP"
```

`ignoredTitleKeywords` 設定を確認してください。

## 4. ベースブランチが監視されていない

デフォルトでは、KodusはデフォルトブランチをターゲットとするPRをレビューします。PRが別のブランチをターゲットにしている場合は、`baseBranches` に追加します：

```yaml
baseBranches:
  - develop
  - staging
```

## 5. レビューの頻度が手動に設定されている

レビューの頻度が**手動**に設定されている場合、Kodusは `@kody start-review` とコメントしたときのみレビューします。

## 6. 自動一時停止がアクティブになっている

複数回素早くプッシュした場合、自動一時停止が発動している可能性があります。クールダウン期間が終わるまで待つか、`@kody start-review` とコメントして手動でトリガーします。

## まだ動作しない場合

PRに `@kody start-review` とコメントして手動レビューをトリガーしてみてください。それでも動作しない場合は、[トラブルシューティングガイド](/how_to_use/ja/code_review/troubleshooting)を確認してください。
