Kodusはオープンソースで、独自のインフラストラクチャにセルフホストできます。これにより、データ、ネットワークアクセス、LLMプロバイダーを完全に制御できます。

## デプロイオプション

### Docker（推奨）

任意のVMにDocker Composeを使用してKodusをデプロイします：

1. Kodusリポジトリをクローンします
2. 環境変数を設定します
3. `docker compose up` を実行します
4. ウェブUIにアクセスしてGitプロバイダーを接続します

### ローカル開発

テストまたは貢献のために：

1. クイックなローカルセットアップのためにオーケストレーターを使用します
2. 単一コマンドですべてのサービスを含みます

## 管理できるもの

- **データ** — すべてのコードとレビューデータがインフラストラクチャ上に留まります
- **LLMプロバイダー** — 任意のプロバイダーまたはセルフホスト型モデルを使用
- **ネットワーク** — ファイアウォールの背後で実行し、外部アクセス不要
- **設定** — すべての設定を完全に制御

## 要件

- DockerとDocker Compose
- APIとワーカーサービスのための十分なコンピュートリソース
- Gitプロバイダー（GitHub、GitLab、Bitbucket）へのネットワークアクセス

完全なデプロイガイドについては、[Dockerでデプロイ](/how_to_deploy/ja/deploy_kodus/generic_vm)を参照してください。
