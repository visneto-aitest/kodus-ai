---
title: "プルリクエストに対してLinearイシューを検証する方法"
description: "AI搭載のビジネスロジック検証を使用して、プルリクエストがLinearイシューに記載された内容を実装していることを確認します。"
---

Linearイシューには詳細な要件、受け入れ基準、スコープの定義が含まれていることが多いです。KodusはPRをLinearイシューと比較して、マージ前のギャップを発見できます。

## 動作方法

1. ワークスペースの[プラグイン](/how_to_use/ja/code_review/plugins)ページで**Linearプラグインを接続**します
2. PRにコメントします：

```
@kody -v business-logic https://linear.app/your-team/issue/TEAM-123
```

3. KodyはLinearイシューのコンテキストを取得し、PRのdiffを読んで、ギャップ分析を作成します

## 得られるもの

- **重要度を持つ調査結果**：MUST_FIX、SUGGESTION、またはINFO
- **要件のトレーサビリティ** — 各調査結果はLinearイシューからの正確なテキストを引用
- **スコープ不一致の検出** — PRがイシューとは異なるドメインで作業している場合
- **検証済み要件** — 正しく実装されたもの

## 自動検証

`business_logic: true` が有効（デフォルト）の場合、プラグインが接続されていれば、Kodyはすべてのレビュー中にLinearからタスクコンテキストを自動的に取得します。

## ヒント

- 最も徹底した検証のために、Linearイシューに明確な受け入れ基準を記述してください
- イシューをリンクしたくない場合は、要件をインラインで貼り付けることもできます：

```
@kody -v business-logic $500以上の注文はキャッシュバッククレジットを発行する必要があります。
割引コードはロイヤルティリワードと組み合わせることはできません。
```

詳細については、[ビジネスロジック検証](/how_to_use/ja/code_review/business_logic_validation)を参照してください。
