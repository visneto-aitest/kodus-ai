コードレビュールールをゼロから書くのは面倒です。Kodusはチームのレビュー履歴を分析し、見つかったパターンに基づいて自動的にルールを提案できます。

## 動作方法

1. **コードレビュー設定** → **Kodyルール** に移動します
2. **Kodyルールを生成** をクリックします
3. Kodusがリポジトリの3ヶ月分のコードレビュー履歴をスキャンします
4. 繰り返されるフィードバックパターンとコーディング設定を識別します
5. チームに合わせたルール提案を生成します

## 次に行うこと

- 生成された提案をレビューします — まだ有効ではありません
- チームのニーズに合ったルールをインポートします
- 有効化する前に必要に応じてカスタマイズします
- インポートされたルールのみがコードレビュー中に適用されます

## 継続的な学習

初期セットアップ後、チームのコーディングスタイルが進化するにつれて、Kodusは**毎週**新しいルール提案を生成します。新しい提案を定期的に確認してください。

## 要件

- リポジトリに少なくとも**3ヶ月**のレビュー履歴が必要
- より良いパターン検出のための定期的なコードレビュー活動
- 適切な権限を持つリポジトリアクセス

詳細については、[ルールの自動生成](/how_to_use/ja/code_review/learning/kody_rules_generation)を参照してください。
