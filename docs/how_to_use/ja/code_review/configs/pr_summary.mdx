---
title: "PR サマリー"
---

Kody の PR サマリー機能を使用すると、簡潔でアクション可能なプルリクエストの説明を自動生成できます。
この設定ガイドでは、チームのワークフローと設定に合わせて機能をカスタマイズする方法を説明します。

## 自動サマリーの有効化

プルリクエストサマリーの自動生成を有効または無効にします：

自動サマリー生成を有効にするオプションをトグルします。

有効にすると、Kody はすべてのプルリクエストのサマリーを生成し、一貫性を保ち、時間を節約します。

## 既存の説明に対する動作

Kody が既存のプルリクエストの説明とどのようにやり取りするかを制御します。3 つのモードから選択できます：

### 置き換え（デフォルト）：

既存の説明を最初から新しく生成されたサマリーで上書きします。

### 追加：

生成されたサマリーを既存の説明の末尾に追加します。

### 補完：

Kody が生成したインサイトと現在の説明を組み合わせ、新しいまたは関連する内容のみをハイライトします。

## カスタム指示

特定のガイドラインを提供することで、Kody がサマリーを生成する方法を微調整します：

カスタム指示テキストフィールドに指示を直接追加します。

**例：**

* 「セキュリティの変更とパフォーマンスへの影響に焦点を当てる。」
* 「変更されたパブリック API をハイライトしながら簡潔にサマリーする。」
