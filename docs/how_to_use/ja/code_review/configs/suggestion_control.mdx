## サジェストの制限

### 制限タイプ

- PR ごと（デフォルト）：プルリクエスト全体での最大サジェスト数
- ファイルごと：ファイルあたりの最大サジェスト数

### 最大サジェスト数

Kody が行えるサジェストの数を設定します：

- 最小：6 サジェスト
- デフォルト：10 サジェスト
- 無制限の場合は 0 に設定

プロのヒント：10 から始めて、チームのフィードバックに基づいて調整します。サジェストが多すぎると圧倒されますが、少なすぎると重要な問題を見逃す可能性があります。

## 重大度レベル

### 最小重大度

Kody が報告する問題を選択します：

- 低/すべて：スタイルの問題、軽微な改善
- 中：一般的なベストプラクティス、潜在的なバグ
- 高：重大な問題、明確なバグ
- クリティカル：セキュリティの問題、主要な障害

## サジェストのグループ化

### 統合コメント（デフォルト）

<img src="https://app.kodus.io/_next/image?url=%2Fassets%2Fimages%2Ffull_grouping.png&w=1080&q=75" alt="統合コメント" />
同様の問題を 1 つのコメントにまとめ、影響を受けるすべてのファイルと行を表示します。以下に最適：

- 繰り返しの問題がある大きな PR
- ディスカッションを集中させる
- 通知ノイズを減らす

### 個別コメント

<img src="https://app.kodus.io/_next/image?url=%2Fassets%2Fimages%2Fminimal_grouping.png&w=1080&q=75" alt="個別コメント" />
各問題に個別のコメント。以下の場合に最適：

- 各場所への直接リンクが必要な場合
- チームが焦点を絞った行固有のディスカッションを好む場合
