---
title: "レビューポリシー"
---

## 目的

Kody のフィードバックをどのように解釈するか、およびリポジトリのマージポリシーとどのように連携するかを明確にします。

## サジェスト対ブロッキング

- サジェスト：デフォルトでは、Kody は評価のための非ブロッキングコメントを投稿します。
- 変更のリクエスト（オプション）：有効にすると、Kody はクリティカルな問題に対してレビューを「変更のリクエスト」としてマークできます。
- 自動承認（オプション）：有効にすると、問題が見つからない場合に Kody は PR を承認します。

<Note>
  しきい値についてチームが合意し、強力な CI が整っている場合にのみブロッキング動作を有効にしてください。
</Note>

## 変更のリクエストをいつ有効にするか

- クリティカルな問題が決して出荷されてはならないセキュリティ重要領域。
- 回帰が高コストなパフォーマンスのホットスポット。
- 明確なコーディング標準と安定したレビュールールを持つリポジトリ。

## 自動承認をいつ使用するか

- 小規模で低リスクの変更（ドキュメント/テスト）。
- 高いテストカバレッジと信頼できる CI を持つチーム。

## 人間の判断が常に重要

Kody はフィードバックを加速しますが、人間の判断の代わりにはなりません。チームはどのサジェストを採用するかとマージのタイミングを決定します。
