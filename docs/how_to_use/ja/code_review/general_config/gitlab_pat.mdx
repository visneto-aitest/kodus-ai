## PAT の概要

Kody が自動コードレビューを実行できるようにするには、GitLab で個人用アクセストークン（PAT）を生成する必要があります。このトークンにより、Kody はリポジトリに安全にアクセスし、制御された方法でコード分析を実行できます。

## PAT（個人用アクセストークン）の生成

トークンを正しく設定するには、以下の手順に従ってください：

1. **GitLab にアクセスし、プロフィール設定に移動します**：

   - [プロフィール設定ページにアクセスするにはここをクリックしてください](https://gitlab.com/-/user_settings/personal_access_tokens)

2. **`Access Tokens` に移動します**：

   - 左サイドバーで下にスクロールして `Access Tokens` を見つけてクリックします。

3. **新しいトークンを追加します**：

   - `Add new token` ボタンをクリックします。

4. **以下の仕様で新しいトークンを設定します**：

   - **トークン名**：`kody_code_review` など識別しやすい名前を選択します。
   - **有効期限**：可能であれば、少なくとも 180 日以上の長い有効期限を選択します。
   - **権限**：以下の権限を設定します：
     - **api**：読み取りアクセス。
     - **read_api**：読み取りアクセス。
     - **read_user**：読み取りアクセス。
     - **read_repository**：読み取りアクセス。
     - **write_repository**：読み取りアクセス。

5. **トークンを確定して保存します**：
   - 権限を設定したら、`Generate token` をクリックします。
   - 生成されたトークンをコピーして安全に保存してください。Kody の設定に必要になります。

## Kody へのトークンの追加

トークンを生成した後、Kody の自動化設定画面に貼り付けます。自動化を有効にしようとすると、設定モーダルが自動的に開きます。
