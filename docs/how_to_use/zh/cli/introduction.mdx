Kodus CLI 直接在终端中审查您的代码变更 — 在代码进入 Pull Request 之前捕获 Bug、安全问题和风格违规。它使用与 Kodus PR 审查相同的引擎，但作用于您的工作目录、暂存区、分支或特定提交。

## 适用人群

- **独立开发者**：在推送前进行预检查。
- **团队**：在所有克隆、IDE 和 CI 作业之间共享一套审查规则，无需每位开发者单独登录。
- **AI 编码代理**（Claude Code、Cursor、Codex、Windsurf）：通过结构化输出实现审查-修复循环。
- **CI/CD 流水线**：根据严重级别阈值使构建失败。

## 输出示例

运行 `kodus review` 对工作目录变更执行审查，会生成结构化报告：受影响的文件、按严重级别分组的问题、可预览并应用的建议修复，以及可用于脚本的退出码。

<img src="/images/kodus_cli_review.jpg" alt="在 CLI 中进行 AI 代码审查"/>

在 `--prompt-only` 模式下，同一审查会变成紧凑、结构化的文本，便于 AI 代理解析并执行。

## 限制

| | 试用（无需账户） | 已认证 |
|---|---|---|
| 每日审查次数 | 5 | 根据您的计划 |
| 每次审查文件数 | 10 | 100 |
| 每文件行数 | 500 | — |
| 每文件最大 diff 大小 | 500 KB | 500 KB |
| 每文件最大内容大小 | 2 MB | 2 MB |

试用限制每天重置。运行 `kodus auth status` 查看当前使用情况。

## CLI 与 Kodus Web

CLI 与 Kodus Web 仪表板共享同一个审查引擎、规则和仓库设置。当您希望在 **PR 尚未创建之前** 审查代码（本地分支、暂存差异或代理循环中），请使用 CLI；Web 仪表板则用于 PR 审查、指标和团队级配置。

通过 `kodus config repo` 和 `kodus rules` 所做的更改会同时应用到两端。
