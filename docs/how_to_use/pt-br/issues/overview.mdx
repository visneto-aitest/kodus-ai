## Resumo Rápido

- **Rastreamento automático**: Captura automaticamente sugestões não implementadas de PRs fechados
- **Resolução inteligente**: Resolve automaticamente quando sugestões são implementadas em PRs futuros
- **Filtragem avançada**: Salve visualizações personalizadas por status, severidade e categoria
- **Zero configuração**: Funciona automaticamente com seu fluxo de trabalho de revisão de código existente

## O que É

O recurso **Kody Issues** rastreia automaticamente todas as sugestões geradas pelo Kody que **não foram implementadas** nos seus pull requests (PRs). Ele funciona como um sonar, fornecendo uma lista em tempo real de melhorias pendentes para garantir que você nunca perca de vista feedback crítico.

## Como Funciona

### 1. Criação de Issues

<Note>
  Issues são criados apenas quando PRs são **fechados** — não quando são abertos ou
  durante a revisão.
</Note>

<Warning>
  Regras do Kody do tipo **PR** não geram issues. Apenas sugestões a nível de arquivo e específicas de código criam issues rastreáveis.
</Warning>

- Quando um PR é **fechado**, o Kody compila todas as sugestões feitas naquele PR que **não foram aplicadas**
- Para cada sugestão não implementada, o Kody cria automaticamente um **Issue** vinculado ao repositório
- Cada issue inclui o contexto original da sugestão, localização do arquivo e nível de severidade

### 2. Resolução Automática

- Se um desenvolvedor implementar a mudança sugerida em um **PR subsequente**, o Kody marca o Issue associado como **Resolvido**
- O Kody corresponde inteligentemente as mudanças implementadas aos issues existentes usando análise de código
- A resolução acontece automaticamente sem intervenção manual

### 3. Gerenciamento Manual

#### Controle de Status

- **Aberto**: Novos issues que precisam de atenção
- **Resolvido**: Marcado automaticamente quando as sugestões são implementadas
- **Descartado**: Marcado manualmente quando as sugestões não são relevantes para sua equipe

#### Ajuste de Severidade

- Cada Issue tem um nível de severidade (`Crítico`, `Alto`, `Médio`, `Baixo`)
- Você pode atualizar os níveis de severidade a qualquer momento para refletir as prioridades da sua equipe
- Use a severidade para focar nas melhorias de maior impacto primeiro

### 4. Filtragem Avançada

<Warning>
  Filtros são ferramentas poderosas — salve visualizações personalizadas para acessar rapidamente os issues
  mais importantes para o seu fluxo de trabalho.
</Warning>

- **Filtros de status**: Aberto, Resolvido, Descartado
- **Filtros de severidade**: Crítico, Alto, Médio, Baixo
- **Filtros de categoria**: Segurança, Desempenho, Estilo de Código, etc.
- **Filtros de repositório**: Foque em projetos específicos
- **Salvar visualizações personalizadas**: Aplique automaticamente seus filtros preferidos

## Por que é Importante

### Para Equipes de Desenvolvimento

- **Nunca perca o controle** de melhorias de código importantes
- **Priorize o trabalho** com base no impacto real e na severidade
- **Aprenda padrões** de sugestões comumente descartadas vs. implementadas

### Para Gerentes de Engenharia

- **Acompanhe tendências** de qualidade de código em repositórios
- **Identifique gargalos** na implementação de sugestões
- **Meça a melhoria** no acompanhamento de revisão de código
